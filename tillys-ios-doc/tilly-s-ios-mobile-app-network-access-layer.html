Tilly's iOS Mobile App Network Access

This document describes the architectural approach for the network access layer and breaks down each major component.

Project Structure

All network access layer code is located in a group called Network. Files are arranged based on area of functionality according to the TILLYS_REST_SERVICE document. The files in the root of the Network group encapsulate their respective web service API and exposes any returned JSON as model objects. These are the core objects that developers will use to interact with the web service backend.

At a high level, areas of responsibility break down into two groups.

  • Routes
    • Files in this group contain the functionality to retrieve the full path for each service call including any parameters.
  • Responses
    • Each service call that returns a JSON response gets mapped into a strongly typed model object. Files are grouped by the web service area of functionality.
      • Cart Page
      • Checkout Details
      • Common Functionalities
      • Express Checkout Details
      • Profile Details

There are some web methods that are repeated across areas of functionality in the API document but are only modeled once in code. For instance, ShowShippingAddress method is found in both CheckoutDetails and ExpressCheckoutDetails but is only modeled in the CheckoutDetails.swift file. Grouping is for organization only and does not impact the scope of classes.
